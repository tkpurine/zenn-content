この記事は、MacBook Air で動かせるローカルLLMを、同じコーディング課題で比べた実測ランキングです。

新しいモデルが出たら同じテストを追加し、ランキングを更新していきます。Zennには月次版として、更新差分と上位モデルの変化を定期的に公開する想定です。

## 結論

2026年6月時点のおすすめは、用途別に分けるとこうです。

| 用途 | おすすめ | 理由 |
|---|---|---|
| 普段使いのローカルLLM | gemma4:e2b | 品質・速度・日本語説明のバランスが最もよい |
| 説明・レビュー重視 | gemma4:e4b | デコレーター説明と修正提案の質が高い |
| Qwen系で品質重視 | qwen2.5-coder:14b | 文脈理解とコード品質が安定 |
| とにかく速い単純生成 | phi4-mini / codegemma:7b | 速いが、バグ検出や説明では信用しすぎない |
| 最高品質の比較基準 | Claude Code | クラウドなのでローカル代替ではないが、品質基準として有用 |

ローカルLLMをCopilot代替として使うなら、まずは `gemma4:e2b` を試すのが現実的です。より丁寧な説明やレビューを期待するなら `gemma4:e4b`、Qwen系なら `qwen2.5-coder:14b` が候補になります。

## タスク別の見どころ

### コードレビュー

```python
def divide(a, b):
    return a / b
```

全モデルがゼロ除算には気づきましたが、その先で差が出ました。

- `qwen2.5-coder:14b` は型チェックまで踏み込んだ
- `gemma4` 系は基本修正と堅牢な実装を分けて説明する傾向があった
- `Claude Code` は型アノテーション、命名、深刻度分類まで整理した

単純なバグ指摘だけなら小型モデルでも十分ですが、実務レビューに近い構造化はクラウド上位モデルや中型ローカルモデルが強いです。

### FizzBuzz生成

単純なコード生成では、多くのモデルが正解しました。差が出たのは、応答速度と出力形式です。

- `codegemma:7b` と `phi4-mini` は非常に速い
- `qwen2.5-coder:14b` は実行結果まで出して丁寧
- `granite3.3:8b` はコードは正しいが、出力形式が標準FizzBuzzからずれた
- `starcoder2:7b` は内部トークンが露出し、チャット用途には不向きだった

### バグ検出

```python
def calculate_average(numbers):
    total = 0
    for n in numbers:
        total += n
    return total / len(numbers)

results = []
for i in range(5):
    results.append(i * 2)
print(calculate_average(results))
```

この課題では、「空リストならゼロ除算になる」という潜在バグと、「このコードでは `results` が5要素になるので今は発動しない」という文脈理解の両方を見ました。

文脈理解まで成功したのは、`qwen2.5-coder:14b`、`qwen3.5:9b`、`gemma4` 系、`Claude Code` です。

一方で、`codegemma:7b` は存在しないバグを指摘し、`phi4-mini` は計算を誤りました。速いモデルほど分析系タスクでは確認が必要です。

### 日本語説明

Pythonのデコレーター説明では、`gemma4:e4b` が最も印象的でした。認証、キャッシュ、ログ、実行時間計測といった実用例を整理し、`*args/**kwargs` の意味まで自然な日本語で説明しました。

`gemma4:e2b` も十分自然で、速度を考えると普段使いに向きます。`qwen3` 系も日本語は流暢ですが、Thinkingに時間がかかる場面があります。

## モデル別メモ

### gemma4:e4b

今回の総合1位。説明の質が高く、コードレビューや学習用途で使いやすいです。平均55秒なので即答感はありませんが、待てるなら最も安定した選択肢です。

### gemma4:e2b

普段使いの第一候補。平均35秒で、文脈理解・コード生成・日本語説明のバランスが良いです。16GB Macでも扱いやすいサイズ感でした。

### qwen2.5-coder:14b

コーディング特化らしく、バグ検出と修正案が安定しています。9GB級なのでMacBook Airでは重めですが、品質重視なら試す価値があります。

### granite4.1:8b

granite3.3から大きく改善しました。バグ検出ではまだ弱さがあるものの、速度と説明品質のバランスは良く、今後の更新を追いたいモデルです。

### qwen2.5-coder:7b

軽くて速い入門枠。単純な生成や軽いレビューには使えますが、文脈を深く読むタスクでは上位モデルに劣ります。

### codegemma:7b / phi4-mini

どちらも速いですが、分析系では不安が残ります。コード生成専用、または回答を人間が必ず確認する前提なら使えます。

### starcoder2:7b

コード補完向けで、チャット指示には合いません。Ollamaで動くモデルでも、用途が「チャット対応」か「補完専用」かは確認が必要です。

### devstral

16GB Macではスワップが大きく、2分経っても出力0トークンでした。24GB以上のMac、またはGPU環境で再評価したいモデルです。

## ローカルLLMを使う理由

ローカルLLMの最大のメリットは、コストよりもプライバシーです。

| 観点 | クラウド型 | Ollamaローカル |
|---|---|---|
| コードの送信先 | 外部サーバ | 自分のPC内 |
| ネット接続 | 必須 | 不要 |
| 社内コード | 規約・社内ルール確認が必要 | ローカルで完結 |
| `.env` や認証情報が近くにある作業 | 注意が必要 | 外部送信なし |
| ランニングコスト | 月額/従量 | セットアップ後は基本0円 |

CopilotやClaude Codeは便利ですが、コードを外部に送れない場面ではローカルLLMが選択肢になります。
