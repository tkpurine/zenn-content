## なぜModelfileが必要か

Ollamaで配布されているモデルは、デフォルトのコンテキスト長（一度に扱える入力+出力のトークン数）が控えめに設定されていることがあります。Aiderで複数ファイルを同時に渡したり、長い指示文＋ソースコードを送ったりすると、デフォルト設定では情報が溢れてしまい、モデルが指示の前半を忘れたような挙動をすることがあります。

この上限を引き上げたカスタムモデルを作るのが `Modelfile` の役割です。既存モデルをベースに、パラメータだけを上書きした派生モデルをローカルに作成できます。

## Modelfileの書き方

テキストファイルとして作成します。ヒアドキュメントでそのまま生成できます。

```sh
cat > Modelfile << 'EOF'
FROM ornith-1.0-9b
PARAMETER num_ctx 32768
PARAMETER temperature 0.1
EOF
```

| 行 | 意味 |
|---|---|
| `FROM ornith-1.0-9b` | ベースにする既存モデル |
| `PARAMETER num_ctx 32768` | コンテキスト長を32,768トークンに拡張 |
| `PARAMETER temperature 0.1` | 出力のランダム性を抑え、コーディング用途向けに決定的な応答にする |

`num_ctx` の値はメモリと相談です。16GB統合メモリの環境では32768（32k）程度が実用的な上限の目安です。メモリに余裕がなければ16384（16k）から試してください。

### temperatureパラメータとは

`temperature` は、モデルが次のトークンを選ぶ際の「ランダム性の強さ」を決めるパラメータです。0〜1程度の範囲で指定します（実装によっては1を超える値も設定できます）。

| 値 | 挙動 |
|---|---|
| `0.0` に近い | 常にもっとも確率の高いトークンを選びやすくなり、同じ入力に対してほぼ同じ出力を返す（決定的） |
| `1.0` に近い、またはそれ以上 | 確率の低いトークンも選ばれやすくなり、出力が多様・創造的になる一方、精度や一貫性は下がりやすい |

文章生成やアイデア出しのようなタスクでは高めの値（0.7〜1.0程度）が好まれますが、**コーディングタスクでは値を下げて決定的な出力にするのが定石**です。構文や論理の正しさが最優先で、表現の多様性はむしろ不要だからです。

### 本書での推奨値：0.1

本書のModelfile例では `0.1` を採用しています。`0.0`（完全に決定的）ではなく、あえて小さな値を残しているのには理由があります。

- **`0.0`にしない理由**：本書は `--auto-test` による自律修正ループを前提としています（7章）。仮に`temperature 0.0`で完全に決定的にしてしまうと、モデルが一度誤った修正を出した場合、同じ入力に対して**毎回まったく同じ誤答を繰り返す**リスクがあります。ごく小さな乱数性を残すことで、再試行のたびにモデルが別の言い回し・別のアプローチを試す余地を残しています
- **大きくしすぎない理由**：0.3〜0.5程度まで上げると、シンタックスの揺れや不要な言い換えが増え、`--edit-format whole`でのファイル全体書き換えの再現性が落ちます。特にAiderの自律ループでは、出力が安定しているほどテスト結果の原因切り分けがしやすくなります

コーディングエージェント全般（GitHub Copilotなど）でも、コード生成には低温設定（0.0〜0.3程度）が一般的に採用されており、`0.1`はその範囲の中でも「ほぼ決定的だが、完全な決定論は避ける」という妥協点にあたります。

もし`--auto-test`を使わず単発の指示だけで使う場合は、より厳密な再現性を優先して`0.0`にしても問題ありません。逆に、同じ指示から複数の実装案を出させたい探索的な用途では、`0.2`〜`0.3`程度まで上げてみるとよいでしょう。

## カスタムモデルの作成

```sh
ollama create ornith-1.0-9b-ctx32k -f Modelfile
```

作成後、`ollama list` に `ornith-1.0-9b-ctx32k` が追加されているはずです。

```sh
ollama list
```

以降、Aiderからはこのカスタムモデル名（`ollama/ornith-1.0-9b-ctx32k`）を指定して利用します。

## つまずきやすいポイント

- **ベースモデル名のtypo**：`FROM` に指定するモデル名は `ollama list` で表示される名前と完全一致させる必要があります。タグ（`:latest` など）を省略した場合の挙動もモデルによって異なるため、迷ったら `ollama list` で正式名称を確認してください
- **既存のカスタムモデルの上書き**：同じ名前で `ollama create` を再実行すると上書きされます。パラメータを試行錯誤する際は問題ありませんが、複数バージョンを比較したい場合は名前を変えて作成してください（例: `ornith-1.0-9b-ctx16k`）
