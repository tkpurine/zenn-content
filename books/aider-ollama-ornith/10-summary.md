## 本書のふりかえり

- **Ollama**は、ローカルLLMのダウンロード・実行・APIサーバー化を担うツール
- **Aider**は、Ollama経由のローカルモデルを含む幅広いLLMバックエンドで自律コーディングができるツール。ローカルモデルには `--edit-format whole` が必須
- **Ornith-1.0-9B**は、DeepReinforce AIが2026年6月に公開した、コーディング特化・MITライセンス（商用利用可）の9Bクラスモデル
- 環境構築は `brew install ollama` → `ollama pull` → `pip install aider-chat` の3ステップ
- `Modelfile` でコンテキスト長を拡張したカスタムモデルを作成できる
- 実測ベンチマークでは、単純タスクから中程度の複雑さのタスクまで速度・品質・信頼性のいずれも高評価。ただし複雑なタスクでは「指示範囲を超えて先の工程まで生成してしまう」「複雑すぎるタスクでは応答不能になる」といった弱点が確認されている

## 運用のチェックリスト

無人・半自動でOrnith-1.0-9Bを運用する際は、以下を徹底してください。

- [ ] 他の重い作業を止めた状態で速度を評価する（バックグラウンド負荷で2.6〜7.5倍遅くなることがある）
- [ ] `--auto-test` の実行前後で `git diff` を取り、テストファイルが改竄されていないか確認する
- [ ] 複雑なタスクは、計画→実装→ドキュメント化のように段階を分けて依頼する
- [ ] 各段階が終わるごとに、指定通りのファイル名・内容で出力されているか確認する
- [ ] 「テストがPASSした」という報告だけで判断せず、テストの中身が空でないか・前提が正しいかも確認する
