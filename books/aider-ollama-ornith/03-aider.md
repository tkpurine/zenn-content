## Aiderの役割

[Aider](https://aider.chat/) は、ターミナルで動くAIペアプログラミングツールです。Claude Codeと似た立ち位置ですが、Anthropicのモデル専用ではなく、OpenAI・Google、そしてOllama経由のローカルモデルまで、幅広いLLMバックエンドを指定して使えるのが特徴です。gitリポジトリ内のファイルを対象に、自然言語の指示でコードを編集させられます。

## なぜAiderを選んだか：OpenCodeからの乗り換え経緯

最初に試したのはAiderではなく、[OpenCode](https://github.com/opencode-ai/opencode) でした。しかし既存ファイルの編集がことごとく不安定で、ローカルモデルとの組み合わせでは実用に耐えませんでした。

```
型アノテーション追加タスク（OpenCodeの場合）
Edit失敗 → リトライ → Edit失敗 → リトライ → Edit失敗
→ 最終手段としてWriteツールでファイル全体を上書き
→ 完了まで約2分
```

原因を調べると、OpenCodeのEditツールは「変更前の文字列（oldString）をファイル内から完全一致で検索し、変更後の文字列に置換する」という方式でした。これはクラウドの大型モデルなら問題になりませんが、**ローカルの中小規模モデルは元のコードを1文字違わず再現するのが苦手**で、検索がことごとく失敗していました。

Aiderには `--edit-format whole` というオプションがあり、これが決め手になりました。

```
whole形式（Aider）:
  モデルが「変更後のファイル全体」を返す → そのまま書き換え → 完全一致検索が不要

diff形式（デフォルト）:
  モデルが「変更前→変更後」の差分を返す → oldStringを検索して置換 → 再現精度が低いと失敗
```

ファイル全体を書き換えさせる分、送受信するトークン量は増えますが、パース失敗によるリトライが激減し、結果的に速く・安定して完了するようになりました。本書のOrnith-1.0-9B活用でも、この `--edit-format whole` を前提にします。

## Aiderの主な機能

- **自然言語での編集指示**：ファイルを指定し、`--message` で指示を渡すだけでコードを書き換えさせられる
- **`--auto-test` による自律修正ループ**：テストコマンドを指定しておくと、失敗時にAider自身が原因を推測して再修正を繰り返す
- **`--no-auto-commits`**：変更を自動でコミットしない設定にでき、レビューしてから手動でコミットする運用が可能
- **`--yes`**：確認プロンプトを自動承認し、非対話（スクリプトやCI）での実行に対応

## 基本的な使い方のイメージ

```sh
aider --model ollama/<モデル名> \
      --edit-format whole \
      --no-auto-commits \
      --yes \
      対象ファイル.py \
      --message "指示内容"
```

各オプションの詳細と、Ornith-1.0-9Bに特化した実行コマンドの雛形は7章でまとめて解説します。まずは次の4章で、本書の主役であるOrnith-1.0-9Bについて詳しく見ていきましょう。
