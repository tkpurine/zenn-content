## はじめに

前回、[rembg](https://github.com/danielgatis/rembg) を使って[背景除去のデモアプリ](https://zenn.dev/tkpurine/articles/bg-remover-rembg)を作りました。動かせたものの「**そもそも背景除去ってどうやって被写体を見分けてるの？**」が気になったので、生成AIに教えてもらいながら質問を重ねて深掘りした記録です。これから背景除去を触る人が、最短で仕組みの全体像をつかめることを目指します。

## rembgとは（まず全体像）

一言でいうと、**画像の背景を自動で消すオープンソースのツール／Pythonライブラリ**です（作者 Daniel Gatis、MITライセンス）。

- 何ができる：被写体だけを残して背景を透明化（RGBAで出力）
- 使い方：Pythonライブラリ / CLI / HTTPサーバー / Docker の4通り
- 中身：学習済みのAIモデルを **ONNX Runtime** で動かす。だからCPUでも動く
- モデルは選べる：汎用 `u2net`、軽量 `u2netp`/`silueta`、人物特化 `u2net_human_seg`、新しめの `isnet`系、Metaの `sam`、高精度な `birefnet`系、最新の `bria-rmbg` など

## 聞いてみた（Q&A）

### Q1. 背景除去って、どういう仕組みで"被写体"を見分けてるの？

**A.** 画像を「ピクセル1個ずつ、前景か背景か」を判定しています。この作業を**セグメンテーション**と呼びます。

モデルに画像を通すと、元画像と同じサイズの**マスク（白黒のグレースケール）**が返ってきます。白いほど「被写体らしい」、黒いほど「背景らしい」という確信度のマップで、rembgはこの白い部分を残し、黒い部分を透明にします。

ポイントは、ルールで判定しているのではなく、**人が前景と背景を塗り分けた大量の正解データから学習している**こと。rembg既定のU2Netは「その画像で一番目立っている主役は何か」を当てる **Salient Object Detection（顕著性物体検出）** という考え方のモデルです。だから雑多な背景でも、手前の主役をちゃんと抜けます。

ただしモデルは「これは猫だ」と意味を理解しているわけではなく、見た目のパターンから「主役っぽさ」を確率で予測しているだけ。これが、主役がはっきりした写真は得意・髪の毛や半透明は苦手、という得手不得手につながります。

### Q2. 複数人が写った集合写真ではどうなる？

**A.** U2Netは「前景か背景か」を付けるだけで、人物Aと人物Bを区別はしません（インスタンス分割ではない）。なので集合写真では、**目立つ人たちをまとめて1つの前景**として残します。「特定の1人だけ」を抜きたいときは、点で対象を指定できる **SAM** モデルの出番です。

### Q3. モデル構造はどんな形？　ResNetなどと比べてどこが良い？

**A.** まず、有名な ResNet とは土俵が違います。下図のとおり、3つは役割からして別物です。

![モデル構造の比較：ResNet / U-Net / U²-Net](/images/rembg/u2net-architecture.png)

- **ResNet**：画像→ラベルを当てる「分類」モデル。出力は1つのラベルで、背景除去に必要な画素単位のマスクは出せない。ただし「残差接続（skip）」で層を深くできるようにした発明は重要。
- **U-Net**：画像を縮小→拡大するU字型で、左右をつなぐ**スキップ接続**で輪郭を復元しながら**マスク**を出す。
- **U²-Net（rembg既定）**：そのU字の**各段を、また小さなU-Net（RSUブロック）にした入れ子構造**。

U²-Netの良いところは3つです。

| 観点 | U-Net | U²-Net |
|---|---|---|
| 1段あたりの視野 | 単一スケール中心 | 各段で多スケール（細部＋全体）を同時に把握 |
| 深さとコスト | 深くすると重くなりがち | 入れ子で"実質的に深い"のに軽量に保てる |
| 事前学習 | バックボーンに依存しがち | 事前学習なしでゼロから学習できる設計 |

特に重要なのが3つ目。多くのモデルはResNet等を「ImageNet分類で事前学習したバックボーン」として土台に使いますが、U²-Netは**最初から"主役を抜く"目的で設計・学習**されています。だから輪郭の質が高く、サイズも抑えられる（標準で約44M、軽量版u2netpは約1.1M／4.7MB）。これが「**CPUでも動くのに実用十分**」につながっています。

### Q4. 学習データと、正解値（ラベル）はどう作る？

**A.** U2Netは **DUTS**（特に学習用の DUTS-TR、約1万枚）という顕著性物体検出の定番データセットで訓練されています。

正解値は、各画像に対する**ピクセル単位の白黒マスク**。**人が手作業で、被写体を白・背景を黒に塗り分けた**もので、これが「答え」です。学習では、モデルが出したマスクと人手マスクのズレが小さくなるよう重みを調整します。アノテーションは境界をなぞる精密な人力作業で、その品質がそのままモデルの「輪郭の上手さ」に直結します。

### Q5. 正解（顕著な対象）は人だけ？　他の物体も含まれる？

**A.** 人だけではありません。DUTSは人物専用ではなく**汎用**で、動物・乗り物・食べ物・植物・日用品・人物など、被写体のカテゴリは非常に幅広いです。正解マスクは「その画像で一番目立つ主役（人とは限らない）」を塗ったもの。だから `u2net` は "general use" とされ、商品・ペット・植物もちゃんと抜けます。

逆に、`u2net_human_seg`（人物）・`u2net_cloth_seg`（服）・`isnet-anime`（アニメ）などは、その対象専用のデータで学習した特化モデルです。

## 使ってみた感想 / 学び

- 「人だから抜けた」のではなく「**主役だから抜けた**」という整理がしっくりきた。
- 髪の毛が苦手なのは、元をたどれば"細部まで完璧に塗られた正解データをどれだけ学べているか"の問題。`isnet`系や `birefnet`系が細部に強いのも納得。
- 次は **アルファマッティング**（輪郭の精細化）や、`birefnet` との精度比較を試したい。

## まとめ

- 背景除去の正体は「画素ごとに前景/背景を当てるセグメンテーション」。
- rembg既定のU²-Netは「U字を入れ子にした」構造で、軽さと精度を両立。
- 精度は結局「人が塗った正解マスクの質と量」に支えられている。
