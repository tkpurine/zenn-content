## はじめに

背景除去デモを [Hugging Face Spaces](https://huggingface.co/spaces) に無料で公開しました。便利だった一方で「無料って実際どこまで？」「人気が出たら課金される？」が気になったので、生成AIに教えてもらいながら深掘りした記録です。個人で無料デモを出したい人向けのまとめです。

## Hugging Face Spacesとは（まず全体像）

機械学習のデモアプリを、サーバー構築なしで公開できるホスティングサービス。Gradio / Streamlit / Docker / 静的HTMLに対応します。**学習（training）ではなく、完成したモデルを動かして見せる＝デモ公開・推論ホスティングが主目的**です（学習はColabや専用環境で行い、成果物をSpacesで公開、が王道）。

## 聞いてみた（Q&A）

### Q1. 無料Spaceはスリープすると毎回待たされる？

**A.** 「毎回」ではなく「**寝た後の最初の1回だけ**」です。無料のCPU Spaceは**48時間アクセスが無いと自動でスリープ**し、誰かがアクセスすると自動で再起動します（起きるまで数十秒〜1分）。一度起きればしばらく速いまま。

注意点として、無料Spaceのディスクは非永続なので、**再起動後の初回推論ではモデル（例：u2netは約176MB）を再ダウンロード**します。これが「初回だけ遅い」の正体。アクセスが多いSpaceはほぼ起きっぱなしなので、待ちはあまり起きません。

### Q2. CPU無料枠で「動く / 動かない」の線引きは？

**A.** 目安は「**モデルが数百MB級＆1枚を数秒以内で処理できるか**」。判断軸は (1) 16GB RAMに収まるか、(2) 1リクエストが数十秒以内か。

| 動く（CPU無料でOK） | 厳しい（GPU/PRO推奨） |
|---|---|
| 背景除去（u2net）、軽い分類・検出 | 画像生成（Stable Diffusion系） |
| OCR、軽いNLP、データ処理 | 大規模LLM（数GB級） |
| 数百MB級の学習済みモデル | 高倍率の超解像、動画・リアルタイム |

無料CPUのスペックは 2 vCPU / 16GB RAM / 50GB（非永続）ディスク。背景除去はちょうど"動く"側でした。

### Q3. プランはどんな種類がある？（Spacesの料金の全体像）

**A.** HFの課金は「**①アカウントのプラン（席・月額）**」と「**②実際に動かすコンピュート（従量課金）**」の2階建てです。

![料金は2階建て：席プランと従量コンピュート](/images/spaces/hf-pricing-tiers.png)

- ①席プラン：Free($0) / PRO($9/月) / Team($20/人・月) / Enterprise($50〜/人・月)。個人なら実質 **FreeかPRO**の2択
- ②コンピュート：Spacesの有料GPU（時間課金）、Inference Endpoints、Inference Providers など。**席プランとは別払い**

ポイントは「**プラン料金は席代。重いモデルを動かすGPU時間は別**」。CPUで軽いデモを見せるだけなら②はほぼ0で、Freeで完結します。

### Q4. ZeroGPUの「1日◯分」と、従量課金のGPUは何が違う？

**A.** 「**共有して使った秒数だけ vs 自分専用で確保している時間ずっと**」の違いです。

| 観点 | ZeroGPU（PROの枠） | 専用GPU（従量） |
|---|---|---|
| GPUの持ち方 | 共有・必要な瞬間だけ借りる | 自分専用に占有 |
| 課金の基準 | 実際に計算した秒数 | 起動している時間（待機中も） |
| 料金 | PRO定額内＋超過$1/10分 | $0.40/時〜 使った時間ぶん |
| 待ち | キュー・コールドスタートあり | なし・即応答 |
| 向く用途 | たまに重い推論を見せるデモ | 常時稼働の本格運用 |

たとえると **ZeroGPU＝カーシェア（乗ってない時は無料）／専用GPU＝レンタカー（停めてても日額が発生）**。

### Q5. デモを公開して人に使われたら、自分が課金される？

**A.** **CPU無料Spaceなら、誰が何回使っても0円**です（アクセス数や訪問者数に応じた課金はありません）。混んでも遅くなるだけで請求は来ません。

ZeroGPUの場合は、**訪問者は払わず、Space所有者（自分）の1日の枠を消費**します。無料枠なら「Space全体で1日約3.5分」を訪問者みんなで共有し、**使い切ったらその日は止まる/待たされるだけ**。勝手に課金は走りません。現金課金が発生するのは「**自分で有料の専用GPUを付けた**」か「**PROで超過クレジット($1/10分)をONにした**」場合だけ。つまり**デフォルトでは"知らぬ間に高額請求"は起きない**設計です。

### Q6. Spacesと、自分のサーバー（VPS）の違いは？

**A.** 「**手軽さのSpaces／自由度のVPS**」。

| 観点 | HF Spaces | 自前VPS |
|---|---|---|
| セットアップ | git pushだけ・管理不要 | OS・SSL・デプロイ・監視を自分で |
| 費用 | 無料枠あり | 月額固定 |
| 常時稼働 | 無料はスリープあり | 常時起動が普通 |
| エコシステム | モデル/自動API/コミュニティ直結 | 何でも自由だが全部自前 |
| 向き | デモ・ポートフォリオ・共有 | 本番サービス・独自要件 |

匿名・手軽・無料で作品を見せて実績を貯める今のフェーズには、Spacesが最適です。

### Q7. そもそも、なぜ人はSpacesに公開する？ PROや課金は何のため？

**A.** 公開する目的は「知名度」だけではなく、いくつかあります。

- **ポートフォリオ・実績・知名度**：ML界隈ではSpacesの公開＝作品集。採用・案件・フォロワーにつながる
- **集客＝"撒き餌"マーケ**：無料の「お試しデモ」で認知を広げ、自社の**有料製品やAPIへ誘導**する。実際、rembgのスポンサーは背景除去APIの「PhotoRoom」で、無料デモ→有料APIという王道パターン
- **研究の可視化・再現**：論文のデモを公開して信頼性・話題性を得る
- **プロトタイプ/MVP**：素早く公開してフィードバックを集める／OSS・教育目的

課金（PRO・有料GPU）は「無料では足りなくなったとき」の話で、目的が分かれます。

- **PRO（$9）**：GPUが要るデモ（画像生成・高倍率超解像など）を自分のSpaceで公開したい個人。ヘビーに実験・公開する人向け
- **有料GPU（従量）**：デモが人気になり、人に常時バリバリ使ってもらう段階。"デモ"というより"サービス"

**結論（個人の準備期間なら）**：目的がポートフォリオ・知名度づくりなら、**無料CPUで十分**。課金が要るのは「GPUデモを出したい」or「常時使われるサービスにしたい」と思ったときだけ。

## まとめ

- Spacesは「デモ公開・推論ホスティング」向き。学習はColab等で。
- 料金は2階建て＝①席プラン＋②従量コンピュート。**CPU無料デモはトラフィックがあっても0円**。
- 重い処理が要るなら ZeroGPU(PRO)、常時運用なら 専用GPU、と段階的に。
- 個人の準備期間は **Freeで十分**。困ったら課金、が正解。
