## はじめに

背景除去デモで巻き毛の写真を試したとき、輪郭にうっすら縁取り（ハロー）が残りました。これを「髪一本一本が自然に透ける」形に直す技術が**アルファマッティング**です。何が嬉しくて、どう動くのかを生成AIに聞きながら深掘りした記録です。

## これをやると何が嬉しい？

ひとことで言うと**境界がきれいになる**こと。具体的には3つです。

1. **ギザギザ・ハローが消える**：二値マスクの硬い切り口だと髪がのこぎり状になったり古い背景色が縁に残る。αで一本一本が半透明に抜けて自然になる
2. **別の背景に合成しても馴染む**：合成式に"正しいα"を使うので、白背景でも別画像でも縁が浮かない
3. **仕上がりがプロっぽい＝実用価値**：人物アイコンやEC商品画像の品質が上がる

## デモで試す

実際に試せるデモを公開しています → **https://huggingface.co/spaces/tkawano/bg-remover**

使い方:
1. 画像をアップロード
2. **Alpha matting** をオン/オフして切り替え（同じ画像で比較できます）
3. 髪がまだ硬ければ **Edge strength**（不明帯の幅）を大きく
4. **Remove background** を押す

> Alpha matting をオンにすると少し遅くなりますが、境界がなめらかになります。

## 全体像

![マッティングの流れ：マスク→trimap→α→合成](/images/alpha-matting/alpha-matting-pipeline.png)

セグメンテーションのマスク（確信度）から trimap を自動生成し、pymatting が α（被覆率）を解き、合成式 `I = α·F + (1−α)·B` で自然に切り抜く、という流れです。

## 聞いてみた（Q&A）

### Q1. マスクのα と、セグメンテーションの出力は何が違う？

見た目はどちらも0〜1ですが意味が別物。**セグメンテーションの出力＝「前景っぽいかの確信度」**、**マッティングのα＝「実際の被覆率（混合比）」**です。α=0.5は"本当に半分が前景・半分が背景"（細い毛が画素を半分覆う）を意味し、合成式に物理的に使えます。前回ハローが出たのは、確信度マップをそのまま透明度に流用したから——真の被覆率でも前景/背景色の分離でもないため、縁に古い背景色が滲みます。

### Q2. trimapは必須？ rembgはどう自動生成する？

古典マッティングは実質trimapが要ります（不定問題を解くため、確実な領域を固定して"不明帯"だけ解く）。rembgは**マスクからtrimapを自動生成**：二値マスクを**収縮(erode)→確実な前景**、**膨張(dilate)→確実な背景**、その**間の帯＝不明領域**。帯の幅は `erode_size`、確実FG/BGの閾値は `foreground/background_threshold` で決まり、不明帯のαを pymatting が解きます。手で塗る必要はありません。

### Q3. 未知数7・式3で"不定"なのに、なぜ解ける？

1画素だけ見ると未知数は α(1)＋F(3)＋B(3)＝**7**、式はRGBの**3**で足りません。解けるのは**画素を独立に解かず、近傍に前提を置いて全体を連立で解く**から。

- **カラーライン仮定**：「小窓の中で前景色・背景色はそれぞれ色空間上の直線に乗る」と仮定すると、αが局所の色の線形結合で表せ、**巨大な疎行列（ラプラシアン）の連立**に落ちて大域的に解ける
- **trimapが境界条件**：確実FG=1・確実BG=0 でαを固定し、不明帯を解く
- ディープ系はこの"前提"をデータから学習して置き換える

### Q4. rembgの `-a`（alpha matting）パラメータ調整

主に4つ。

- `alpha_matting=True`（ON）
- `alpha_matting_foreground_threshold`（既定240前後）：これ以上＝確実な前景
- `alpha_matting_background_threshold`（既定10前後）：これ以下＝確実な背景
- `alpha_matting_erode_size`（既定10前後）：**不明帯の幅**

勘所：**髪がまだ硬い/ハロー → `erode_size` を大きく**。**被写体が削れる/透けすぎ → `erode_size` を小さく、閾値を上げる**。反復で詰める。トレードオフは**CPUで遅くなる**こと。デモではトグルで出すのが定石です。

### Q5. 古典（pymatting）と ディープ（MODNet等）はどっち？

| | 古典（pymatting / rembg -a）| ディープ（MODNet 等）|
|---|---|---|
| 仕組み | マスク＋画像から疎行列を解く | 学習済みモデルが推定 |
| 品質（髪・毛皮）| そこそこ（trimap品質に依存）| 高い・複雑な境界に強い |
| 依存・コスト | 軽い・モデル不要・CPU向き | モデルが要る・GPU推奨 |
| trimap | 要（自動生成）| 不要なものが多い |

軽く底上げ → pymatting（rembgの `-a`）。髪で本気の品質・人物中心 → MODNet等。rembg内なら `birefnet` 系の高精度モデルに替える手もあります。

## まとめ

- マッティングは「境界の混ざり」を**α（被覆率）**として解く技術。マスクの確信度とは意味が違う。
- 古典は **trimap＋色の滑らかさ**という前提で、不定問題を疎な線形問題に変換して解く。
- rembgは `-a` で手軽に試せる（`erode_size` が要）。本気の髪なら MODNet 等のディープへ。
- 前回の巻き毛のハローは、まさにこれで改善できる。
