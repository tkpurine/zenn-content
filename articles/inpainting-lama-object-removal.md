## はじめに

これまで「選ぶ（SAM）」「拡大（Real-ESRGAN）」「顔を直す（GFPGAN）」を作ってきました。次は**消す**——写真から不要な物体を消して背景を自然に埋める **Inpainting（インペイント／穴埋め）** です。CPU無料Spaceで動く **LaMa** でデモを作ったので、その記録と調べたことをまとめます。

## Inpaintingとは（まず全体像）

**画像の一部（マスクした領域）を、周囲と自然につながるように埋め直す**技術です。用途は「不要な人・物・電線・傷を消す」「透かしやゴミを除去する」など。埋め方には大きく3つの流派があります。

![Inpaintingの3つの流派：古典・LaMa・拡散](/images/inpainting/inpaint-families.png)

今回使う **LaMa**（Large Mask inpainting）は、**物体を消して背景をもっともらしく生成**するのが得意で、CPUでも数秒で動く"ちょうどいい"選択です。

## 「選ぶ → 消す」：マスクさえ作ればいい

LaMaへの入力は**「画像＋2値マスク（白＝消す領域）」だけ**。マスクの作り方は問いません。今回は手描きブラシですが、**これまで作ったSAMの切り抜きをそのまま流し込めば「クリックで選ぶ→LaMaで消す」**という物体除去パイプラインになります。

## 聞いてみた（Q&A）

### Q1. なぜ穴を"自然に"埋められるの？

LaMaは**大量の「劣化（穴あき）画像→元画像」ペアで学習**しており、「この周辺ならこういう背景が続くはず」を推論できます。肝は **FFC（Fast Fourier Convolution／高速フーリエ畳み込み）**。通常の畳み込みは近傍しか見ませんが、FFCは**周波数領域を経由して画像全体を一度に参照**します。つまり**最初の層から"大域的な文脈"を持てる**ので、穴の反対側にある模様やパターンの周期性まで踏まえて、破綻しない背景を1回の推論で作れます。

### Q2. OpenCVの `cv2.inpaint`（古典）と何が違う？

古典（Telea / Navier-Stokes）は**周囲の色を穴の内側へ拡散させる数式ベース**。小さな傷や単純な背景には速くて十分ですが、**大きい穴や複雑な構造は苦手**（のっぺりボケる／構造が繋がらない）。LaMaは**学習で"背景らしさ"を獲得**しているので、大きなマスクでも模様や輪郭を再構成できます。「傷消し＝古典、物体除去＝LaMa」が目安です。

### Q3. なぜ"大きなマスク"に強いの？

まさにFFCの**大域受容野**のおかげです。局所畳み込みだけのネットワークは、穴が受容野より大きいと「穴の中心付近で手がかりが尽きる」ため崩れます。FFCは全体を見られるので、**穴が大きくても周囲全体の情報で埋められる**。LaMaが「Large Mask」を名乗る理由がここです。

### Q4. 拡散インペイント（Stable Diffusion）との違い・使い分けは？

拡散インペイントは**テキスト指示に沿って"新しい物体を描き足す"**のが得意（「ここに花を追加」など創造的な差し替え）。反面、**重く・不安定で・意図せぬ改変**も起きがち、CPUでは遅い。LaMaは逆に**「消して背景で埋める」に特化**し、軽く安定。**"消す"ならLaMa、"別物を描く"なら拡散**、と役割が違います。

### Q5. マスクはどう作る？ 実装で工夫した点

Gradio 5 の `gr.ImageEditor` でブラシ塗りさせ、その**塗った層(layer)のアルファから2値マスクを作成**しています。ただしバージョンによって塗りが `layers` に入らないことがあるため、**入っていなければ `composite` と `background` の差分からマスクを復元**するフォールバックも入れました。さらに **「Grow mask（膨張）」スライダー**で数px広げ、**物体のフチや影を取りこぼさない**ようにしています（`cv2.dilate`）。

```python
def _extract(editor):
    bg = editor["background"].convert("RGB")
    mask = np.zeros(bg.size[::-1], np.uint8)
    for ly in (editor.get("layers") or []):          # 塗った層のalpha
        a = np.array(ly.convert("RGBA"))[:, :, 3]
        mask = np.maximum(mask, (a > 0).astype(np.uint8) * 255)
    if mask.max() == 0:                               # フォールバック：合成との差分
        comp = np.array(editor["composite"].convert("RGB")).astype(int)
        diff = np.abs(comp - np.array(bg).astype(int)).sum(2)
        mask = (diff > 30).astype(np.uint8) * 255
    return bg, mask
```

### Q6. CPUで実用になる？

なります。LaMaは**1回の順伝播**で埋めるだけ（拡散のような反復がない）なので、CPUでも中サイズ画像を数秒で処理できます。初回だけ `big-lama` モデルをダウンロードします。

## 実際に作ってわかったこと

- **マスクの取り方が地味に鬼門**：ImageEditorは「塗った層」を直接くれないことがある → `layers` 優先＋`composite`差分のフォールバックが安定
- **膨張(dilate)が効く**：物体ちょうどに塗っても縁や影が残りがち → 数px広げると自然に消える
- **LaMaは"消す"専用**：新しい物を描き足すのは範囲外。そこは拡散インペイントの仕事
- **SAMと直結できる**：マスク元を差し替えるだけで「クリックで選ぶ→消す」に発展

## まとめ

- Inpaintingは**マスク領域を自然に埋め直す**技術。流派は古典／LaMa／拡散。
- **LaMaはFFCの大域受容野**で、大きな穴でも背景を破綻なく1回で埋める。CPUで実用的。
- **入力は画像＋2値マスクだけ**。手描きでもSAMでもよく、「選ぶ→消す」に繋がる。
- **"消す"はLaMa、"描き足す"は拡散**、と使い分ける。
